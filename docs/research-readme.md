# 研究过程文档

本目录记录 HAB 项目从零到一的完整探索与设计演进过程，供深度理解项目设计决策。

## 文档索引

| 文档 | 内容 |
|------|------|
| `RESEARCH_LOG.md` | **工具矩阵探索研究日志** — 从 HA/UniFi API 能力面探查开始，经历 8 个阶段的逐步推导，最终收敛到 8+1 工具矩阵。记录每个阶段的设计决策、采样数据、替代方案与拒绝理由 |
| `prompt_log.md` | **提示词演进日志** — LLM 系统提示词、工具定义和输出规范的版本演变（V1→V4）。记录每次触发变更的原因、实测 Token 消耗对比、行为变化分析 |

## 阅读建议

**新读者**推荐阅读顺序：

1. 先看 `README.md`（项目主文档）建立全局认知
2. 读 `RESEARCH_LOG.md` 阶段 1-4：理解工具矩阵如何从 API 实际能力中推导
3. 读 `prompt_log.md` V1→V2：理解 LLM 职责的一次重大重构
4. 读 `RESEARCH_LOG.md` 阶段 5-8：理解抽象层级演进、DPI 扩展、纯数据原则

**设计亮点速览**：

- 阶段 5：`get_sensor_details` → `get_room_status` — 从"API 裸封装"到"语义聚合"的跃迁
- 阶段 8：工具输出 53% 瘦身 — 纯数据原则 + type→attrs 白名单
- prompt_log V4：方向约束引入后 Token 消耗 -57%（48K→21K）
